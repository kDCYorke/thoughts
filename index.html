Step 1

Install VS Code

VS Code is the code editor used by millions of developers worldwide. It works great on both your Chromebook and Mac — and they'll feel almost identical once set up.

💻

Ari's Chromebook (Linux)

  • Make sure Linux is enabled: Settings → Advanced → Developers → Linux development environment → Turn on
  • Go to code.visualstudio.com and download the .deb (ARM or x64 — match your Chromebook)
  • Open the Files app, find the download, right-click → Install with Linux
  • Or in the Linux terminal: sudo apt install ./code*.deb
  • Launch from the app launcher — it appears under "Linux apps"
🍎

Ari's Mac (at home)

  • Go to code.visualstudio.com and download the Mac version
  • Open the .zip, drag Visual Studio Code to your Applications folder
  • Launch it — macOS may ask to confirm you want to open it. Click Open.
  • Bonus: install Python from python.org if not already installed (type python3 --version in Terminal to check)

Install Extensions — Do this on both computers

1

5

You're ready! 🎉

Open a folder for your project (File → Open Folder), create a new .py file, and start coding. The file explorer on the left shows all your project files.

⚠️ Codespaces note: GitHub Codespaces lets you run VS Code in a browser (no install needed). It works for most Python, but Pygame games need a real screen to display on — so for game dev, use the Linux subsystem on the Chromebook or the Mac instead.
Step 2

Keep Files in Sync

You'll be coding on two computers. GitHub is the pro-developer way to keep everything in sync — and VS Code makes it easy with buttons, no terminal needed for the basics.

1

Create a free GitHub account

Go to github.com and sign up. It's free. GitHub is where your code lives in the cloud — like Google Drive, but for code.

2

Create a new repository

Click the + button → "New repository". Give it a name like my-game. Make it Private if you want. Click "Create repository".

A repository (or "repo") is like a project folder that GitHub tracks every change in.

3

Connect VS Code to GitHub

Click the Source Control icon (looks like a branch, left sidebar). Click "Clone Repository" and sign in to GitHub when asked. Choose your repo and pick a folder to save it.

4

First Project

🧮 Build a Calculator

Before diving into games, Ari's first project is a GUI calculator. It's the perfect starter — small enough to finish, but teaches you the core ideas: windows, buttons, layout, and handling user input.

🧩

What you'll learn

Creating windows, laying out buttons in a grid, handling click events, and updating a display — all the building blocks of any GUI app.

✨

Why CustomTkinter?

Same simplicity as tkinter but looks modern out of the box. A calculator you can actually show people without apologising for how it looks.

🎮

Then: games!

Once the calculator is done, the jump to Pygame Zero is natural — you already understand windows, events, and functions. Games are next.

⚡ Install first: open the VS Code terminal and run pip install customtkinter

Full Calculator — copy this into calculator.py and run it

pythonimport customtkinter as ctk

class Calculator(ctk.CTk):
    def __init__(self):
        super().__init__()
        self.title("Ari's Calculator")
        self.geometry("320x480")
        self.resizable(False, False)
        self.expression = ""

        # ── Display ──────────────────────────────────
        self.display = ctk.CTkLabel(
            self, text="0", font=("Arial", 36), anchor="e"
        )
        self.display.grid(row=0, column=0, columnspan=4,
                          padx=16, pady=20, sticky="ew")

        # ── Button layout ────────────────────────────
        buttons = [
            ["C", "±", "%",  "÷"],
            ["7", "8", "9",  "×"],
            ["4", "5", "6",  "−"],
            ["1", "2", "3",  "+"],
            ["0", ".", "="],   # "0" spans 2 columns
        ]

        for r, row in enumerate(buttons):
            col = 0
            for label in row:
                colspan = 2 if label == "0" else 1
                btn = ctk.CTkButton(
                    self, text=label,
                    width=60, height=60,
                    font=("Arial", 20),
                    corner_radius=8,
                    command=lambda l=label: self.press(l)
                )
                btn.grid(row=r+1, column=col, padx=5, pady=5,
                         columnspan=colspan, sticky="ew")
                col += colspan

        self.columnconfigure([0,1,2,3], weight=1)

    def press(self, key):
        if key == "C":                          # clear
            self.expression = ""
            self.display.configure(text="0")

        elif key == "=":                         # calculate
            try:
                expr = (self.expression
                        .replace("×", "*")
                        .replace("÷", "/")
                        .replace("−", "-"))
                result = str(eval(expr))
                self.display.configure(text=result)
                self.expression = result
            except:
                self.display.configure(text="Error")
                self.expression = ""

        else:                                     # any other key: append it
            self.expression += key
            self.display.configure(text=self.expression)

app = Calculator()
app.mainloop()

Pro Tips

Tips for Coding Success

Things that'll save you time and frustration.

📁

Use the File Explorer

In VS Code, the sidebar on the left shows all your project files. Click any file to open it. Multi-file projects are easy once you get used to this.

📸

Commit early, commit often

Don't wait until your project is "finished" to commit. Save a snapshot every time you get something working. Write short messages: "add jumping", "fix crash on exit".

🔴

Read error messages top to bottom

When your code crashes, Python gives you an error message. The last line is usually the most useful one. Read it, then Google it.

🔍

Google is your superpower

Copy the error text, paste it into Google, add "python". You'll usually find a Stack Overflow answer that solves it. Every developer does this — even pros.

🖥️

Use the integrated terminal

Press Ctrl+` to open a terminal right inside VS Code. Run your code with python game.py or pgzrun game.py without switching windows.

🧩

Change one thing at a time

When adding a new feature, add it, test it, then commit. If you change five things and something breaks, it's hard to find which change caused the problem.

💡

print() is your debugger

When something isn't working, add print(variable_name) to see what value it has. It sounds basic — but it works even for experienced developers.

🚀

Start small, add features

Get a window open first. Then make something move. Then add controls. Building small pieces that each work is much easier than planning the whole game upfront.
